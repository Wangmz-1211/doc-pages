01 Advanced Language Detection 高级语言检测 高度な言語検出

Beyond basic detection, the service handles mixed languages and ambiguous content. It analyzes the script, sentence structure, and vocabulary distribution.

除了基本检测之外，该服务还可以处理混合语言和模棱两可的内容。它分析脚本、句子结构和词汇分布。

基本的な検出に加えて、このサービスは混合言語や曖昧なコンテンツを処理します。スクリプト、文構造、語彙分布を分析します。

Handling Ambiguity

处理歧义

曖昧さの処理

Some words exist in multiple languages (e.g., "impossible" in English and French). The country_hint parameter guides the model.

有些单词存在于多种语言中（例如，英语和法语中的“impossible”）。country_hint 参数可指导模型。

一部の単語は複数の言語に存在します（例：英語とフランス語の「impossible」）。country_hint パラメータがモデルをガイドします。

  • Input: "impossible" (country_hint="US") → English
  • Input: "impossible" (country_hint="FR") → French
  • Input: ":-)" → Unknown (Score: NaN)

Mixed Language Content

混合语言内容

混合言語コンテンツ

The service returns the predominant language. It does not separate the text into segments by language.

该服务返回主要语言。它不会按语言将文本分割成段。

サービスは主要な言語を返します。言語ごとにテキストをセグメントに分割することはありません。

Warning: 警告： 警告： If the text is widely mixed (50/50), the score will be lower, but it will still pick one ISO code. 如果文本混合广泛 (50/50)，分数会较低，但它仍会选择一个 ISO 代码。 テキストが広く混在している場合（50/50）、スコアは低くなりますが、それでも1つのISOコードを選択します。

02 Sentiment Analysis & Opinion Mining 情感分析与意见挖掘 感情分析とオピニオンマイニング

Opinion Mining (Aspect-Based Sentiment)

意见挖掘（基于方面的情感）

オピニオンマイニング（アスペクトベースの感情）

Standard sentiment gives a score for the whole document. Opinion Mining breaks it down by identifying targets (nouns) and assessments (adjectives). This is critical for product reviews.

标准情感分析给出整个文档的分数。意见挖掘通过识别目标（名词）和评估（形容词）将其细分。这对于产品评论至关重要。

標準的な感情分析は、ドキュメント全体のスコアを提供します。オピニオンマイニングは、ターゲット（名詞）と評価（形容詞）を特定することで詳細に分析します。これは製品レビューにとって重要です。 

from azure.ai.textanalytics import TextAnalyticsClient
from azure.core.credentials import AzureKeyCredential

# Initialize client
endpoint = "https://<your-resource>.cognitiveservices.azure.com/"
key = "<your-key>"
client = TextAnalyticsClient(endpoint=endpoint, credential=AzureKeyCredential(key))

documents = [ 
    "The room was great, but the staff was rude. The breakfast was cold." 
]

# Enable Opinion Mining
result = client.analyze_sentiment(documents, show_opinion_mining=True)
docs = [doc for doc in result if not doc.is_error]

for doc in docs:
    print(f"Document Sentiment: {doc.sentiment}") # Mixed
    for sentence in doc.sentences:
        print(f"Sentence: {sentence.text}")
        for mined_opinion in sentence.mined_opinions:
            target = mined_opinion.target
            print(f"  Target: '{target.text}' ({target.sentiment})")
            for assessment in mined_opinion.assessments:
                 print(f"    Assessment: '{assessment.text}' ({assessment.sentiment})")

# Output Example:
# Sentence: The room was great, but the staff was rude.
#   Target: 'room' (positive) -> Assessment: 'great' (positive)
#   Target: 'staff' (negative) -> Assessment: 'rude' (negative)

03 NER & PII Detection Policies NER 与 PII 检测策略 NER と PII 検出ポリシー

PII (Personally Identifiable Information)

PII (个人身份信息)

PII (個人を特定できる情報)

Essential for compliance (GDPR, HIPAA). The service can identify entities and redact them in the output.

对于合规性（GDPR、HIPAA）至关重要。该服务可以识别实体并在输出中编校（掩盖）它们。

コンプライアンス（GDPR、HIPAA）に不可欠です。サービスはエンティティを識別し、出力でそれらを編集（秘匿化）できます。

  • General: Email, Phone, Address, IP Address.
  • Finance: Credit Card, SWIFT code, IBAN.
  • Government: SSN (US), Driver's License, Passport.

Entity Linking

实体链接

エンティティリンク

Disambiguates entities by linking them to a knowledge base (Wikipedia). It provides the url and dataSource.

通过将实体链接到知识库（维基百科）来消除实体的歧义。它提供 url 和 dataSource。

エンティティをナレッジベース（ウィキペディア）にリンクすることで、エンティティの曖昧さを解消します。url と dataSource を提供します。

"Venus" → Linked to Planet (Context: Space)
"Venus" → Linked to Goddess (Context: Mythology)

A specialized model trained on medical data to extract insights from unstructured clinical documents like discharge summaries, doctor's notes, and trial protocols.

一种在医疗数据上训练的专用模型，用于从非结构化临床文档（如出院小结、医生笔记和试验方案）中提取见解。

医療データでトレーニングされた特殊なモデルで、退院サマリー、医師のメモ、治験実施計画書などの非構造化臨床ドキュメントから洞察を抽出します。

05 Asynchronous Batch Processing 异步批处理 非同期バッチ処理

For processing millions of documents, synchronous APIs (REST/SDK single calls) are inefficient and subject to throttling. Use the Analyze Actions API (/analyze-text/jobs).

对于处理数百万个文档，同步 API（REST/SDK 单次调用）效率低下且容易受到限制。请使用 Analyze Actions API (/analyze-text/jobs)。

数百万のドキュメントを処理する場合、同期 API（REST/SDK 単一呼び出し）は非効率的で、スロットリングの対象になります。Analyze Actions API (/analyze-text/jobs) を使用してください。

  • Submit a batch of up to 25 documents (in list) or pointer to blob storage (containers).
  • 提交一批最多 25 个文档（在列表中）或指向 Blob 存储（容器）的指针。
  • 最大25のドキュメントのバッチ（リスト内）または Blob Storage（コンテナ）へのポインタを送信します。
  • Poll for status (NotStarted -> Running -> Succeeded).
  • 轮询状态（未开始 -> 运行中 -> 成功）。
  • ステータスをポーリングします（未開始 -> 実行中 -> 成功）。
  • Results stored for 24 hours.
  • 结果保存 24 小时。
  • 結果は24時間保存されます。

06 Docker Containers (On-Prem / Edge) Docker 容器 (本地 / 边缘) Docker コンテナ (オンプレミス / エッジ)

Text Analytics can be deployed as a Docker container. This is useful for security/compliance (data doesn't leave your network) or low latency scenarios.

文本分析可以部署为 Docker 容器。这对于安全/合规（数据不离开您的网络）或低延迟场景非常有用。

Text Analytics は Docker コンテナとしてデプロイできます。これは、セキュリティ/コンプライアンス（データがネットワーク外に出ない）や低遅延のシナリオに役立ちます。

要求：

要件：

07 Security & Privacy 安全与隐私 セキュリティとプライバシー

  • Managed Identity: Eliminate hardcoded keys by using Azure AD (RBAC roles like "Cognitive Services User").
  • 托管标识： 通过使用 Azure AD（“认知服务用户”等 RBAC 角色）消除硬编码密钥。
  • マネージド ID： Azure AD（「Cognitive Services ユーザー」などの RBAC ロール）を使用して、ハードコードされたキーを排除します。
  • Virtual Networks (VNET): Restrict access to specific IP ranges or VNET subnets.
  • 虚拟网络 (VNET)： 限制对特定 IP 范围或 VNET 子网的访问。
  • 仮想ネットワーク (VNET)： 特定の IP 範囲または VNET サブネットへのアクセスを制限します。
  • Customer Managed Keys (CMK): Encrypt the data at rest using your own keys in Key Vault.
  • 客户管理的密钥 (CMK)： 使用 Key Vault 中您自己的密钥对静态数据进行加密。
  • カスタマーマネージドキー (CMK)： Key Vault 内の独自のキーを使用して、保存データを暗号化します。
